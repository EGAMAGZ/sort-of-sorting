# Sort of Sorting

> A curated collection of sorting algorithms that range from "reasonably
> efficient" to "what were they thinking?"

## What is this?

Welcome to **Sort of Sorting**, where we celebrate the magnificent spectrum of
algorithmic inefficiency. This is a playground for:

- Learning how sorting algorithms work under the hood
- Joking about algorithms that make your computer question its life choices
- Appreciating the sheer intellectual gymnastics required to make something so
  slow
- Proving that computer science education was not wasted on us

Because why just use `Array.prototype.sort()` when you could spend hours
implementing a sorting algorithm that takes longer than the heat death of the
universe?

## The Collection

| Algorithm  | Time Complexity | Our Verdict                                             |
| ---------- | --------------- | ------------------------------------------------------- |
| Bogo Sort  | O(n!)           | "I'll just keep shuffling until the universe ends"      |
| Bozo Sort  | O(n! × n)       | "Bozo was definitely not a computer scientist"          |
| Sleep Sort | O(max(items))   | "Time is just a suggestion anyway"                      |
| Slow Sort  | O(n^(log n/2))  | "Philosophical sorting for the contemplative developer" |
| Trump Sort | O(1)            | "The array is already sorted, believe me"               |

## Contributing

Found another gloriously inefficient sorting algorithm? Have a witty remark to
add to an existing JSDoc? This is exactly the kind of project that thrives on
community participation.

We welcome:

- New sorting algorithms (as long as they're appropriately terrible)
- Improved documentation (because even bad code deserves good docs)
- Memes (the algorithms deserve to be memes)

Open a PR, file an issue, or just yell into the void. We hear the void.

## Philosophy

This project exists because:

1. **Education**: Understanding why algorithms are inefficient is just as
   important as understanding why they're efficient. Plus, writing bad code is
   the best way to appreciate good code.

2. **Humor**: The best way to learn a complex topic is to laugh at it first. And
   what could be funnier than an algorithm that sorts by hoping for the best?

3. **Appreciation**: Next time you call `.sort()`, take a moment to appreciate
   that Timsort doesn't require you to wait until the heat death of the
   universe.
